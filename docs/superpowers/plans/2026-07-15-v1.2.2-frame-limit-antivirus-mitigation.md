# v1.2.2 帧率限制与误报缓解实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 发布 v1.2.2，为桌面 Renderer 增加默认 60 FPS、30–240 FPS 与无限制模式，并以身份元数据、隔离诊断和独立 ZIP 哈希文件低成本缓解安全软件误报问题。

**Architecture:** 新增无 Qt 依赖的帧率领域模型和 Windows 高精度调度器，Qt Core 与 Renderer 共享同一规范化接口；高级设置只持久化配置，Renderer 下次启动时读取并应用。误报缓解不控制任何安全软件，只增强 VERSIONINFO、区分 `blackhole.exe` 隔离场景，并在正式打包后生成 ZIP 哈希 sidecar 和可直接粘贴的 GitHub Release 说明。

**Tech Stack:** C++17、Win32/QPC/Waitable Timer、OpenGL/WGL、Qt 6.11.1/QML、CMake/CTest、PowerShell、GitHub Release。

## Global Constraints

- 文档和新增用户文本使用中文，源文件使用 UTF-8。
- `frameRateLimit=0` 表示无限制，`30–240` 表示目标 FPS，默认值固定为 `60`。
- QML 滑块范围为 `30–241`，UI 值 `241` 与 Core 值 `0` 双向映射。
- 帧率限制只控制 `blackhole.exe --render`，不控制 Qt 实时预览，不改变动画、鼠标物理或移动速度时间基准。
- 不关闭安全软件、不写排除项、不恢复隔离文件、不上传 VirusTotal、不做厂商申诉、不伪造签名。
- 不使用 UPX、加壳、混淆、随机填充或工具链迁移。
- 版本唯一来源同步为 `1.2.2`，最终文件名为 `BlakholeUI-v1.2.2-windows-x64.zip`。
- 收款码进入本地 Release/ZIP，但不得被 Git 跟踪；ZIP 和 sidecar 也不进入 Git。

```cpp
#include "frame_limiter.h"

#include <cmath>
#include <cstdlib>

namespace {

void require(bool condition)
{
    if (!condition) std::abort();
}

void requireNear(double actual, double expected)
{
    require(std::fabs(actual - expected) < 0.000001);
}

} // namespace

int main()
{
    require(NormalizeFrameRateLimit(0) == 0);
    require(NormalizeFrameRateLimit(1) == 30);
    require(NormalizeFrameRateLimit(30) == 30);
    require(NormalizeFrameRateLimit(60) == 60);
    require(NormalizeFrameRateLimit(240) == 240);
    require(NormalizeFrameRateLimit(241) == 240);

    const FrameDeadlineDecision unlimited = AdvanceFrameDeadline(0, 10.0, 8.0);
    require(!unlimited.shouldWait);
    requireNear(unlimited.nextDeadlineSeconds, 10.0);

    const FrameDeadlineDecision first = AdvanceFrameDeadline(60, 10.0, 0.0);
    require(first.shouldWait);
    requireNear(first.nextDeadlineSeconds, 10.0 + 1.0 / 60.0);

    const FrameDeadlineDecision steady = AdvanceFrameDeadline(
        60, 10.016, first.nextDeadlineSeconds);
    require(steady.shouldWait);
    requireNear(steady.nextDeadlineSeconds, first.nextDeadlineSeconds + 1.0 / 60.0);

    const FrameDeadlineDecision late = AdvanceFrameDeadline(60, 11.0, 10.02);
    require(late.shouldWait);
    requireNear(late.nextDeadlineSeconds, 11.0 + 1.0 / 60.0);
    return 0;
}
```

- [ ] **Step 3: 实现纯逻辑接口**

`src/frame_limiter.h` 定义：

```cpp
#pragma once

#include <windows.h>

inline constexpr int kDefaultFrameRateLimit = 60;
inline constexpr int kMinimumFrameRateLimit = 30;
inline constexpr int kMaximumFrameRateLimit = 240;

int NormalizeFrameRateLimit(int value);

struct FrameDeadlineDecision {
    bool shouldWait = false;
    double nextDeadlineSeconds = 0.0;
};

FrameDeadlineDecision AdvanceFrameDeadline(int frameRateLimit,
                                           double nowSeconds,
                                           double previousDeadlineSeconds);

class FrameLimiter {
public:
    explicit FrameLimiter(int frameRateLimit);
    ~FrameLimiter();
    FrameLimiter(const FrameLimiter &) = delete;
    FrameLimiter &operator=(const FrameLimiter &) = delete;

    void setFrameRateLimit(int frameRateLimit);
    int frameRateLimit() const;
    void reset();
    void wait();

private:
    static double nowSeconds();
    void waitUntil(double deadlineSeconds);

    int m_frameRateLimit = kDefaultFrameRateLimit;
    double m_nextDeadlineSeconds = 0.0;
    HANDLE m_waitableTimer = nullptr;
};
```

`NormalizeFrameRateLimit()` 保留 `0`，其余值夹到 `30–240`。`AdvanceFrameDeadline()` 使用 `1.0 / limit`；旧截止时间无效或落后超过一个周期时从当前时间重新开始，否则累加一个周期。

- [ ] **Step 4: 实现 Windows 等待器**

`FrameLimiter` 构造时优先使用：

```cpp
constexpr DWORD kCreateWaitableTimerHighResolution = 0x00000002;
m_waitableTimer = CreateWaitableTimerExW(
    nullptr, nullptr, kCreateWaitableTimerHighResolution, TIMER_ALL_ACCESS);
if (!m_waitableTimer) {
    m_waitableTimer = CreateWaitableTimerW(nullptr, TRUE, nullptr);
}
```

`waitUntil()` 将剩余秒数转换为负的 100 ns 相对时间，调用 `SetWaitableTimer()` 和 `WaitForSingleObject()`；Timer 不可用时，循环使用 QPC 和短 `Sleep(1)`。析构必须关闭句柄。
