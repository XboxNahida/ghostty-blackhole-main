# v1.2.1 版本同步实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将当前应用、两个 EXE、Release 元数据、安全契约和 ZIP 文件名完整同步为 `1.2.1`。

**Architecture:** `cmake/AppVersion.cmake` 继续作为唯一构建版本源；发布脚本和版本验证脚本同步使用同一目标版本。历史更新状态测试数据不修改。

**Tech Stack:** CMake、Windows VERSIONINFO、PowerShell、Qt 6、CTest。

## Global Constraints

- 当前产品版本固定为 `1.2.1`，Windows 文件版本固定为 `1.2.1.0`。
- Release 标签固定为 `v1.2.1`。
- 不机械替换 `tests/update_checker_state_tests.cpp` 中的历史 `v1.2.0` 测试样例。
- 收款码进入本地 ZIP，但不进入 Git。
- 最终 ZIP 文件名为 `BlakholeUI-v1.2.1-windows-x64.zip`。

---

### Task 1: 版本源与发布契约同步

**Files:**
- Modify: `cmake/AppVersion.cmake`
- Modify: `package_release.ps1`
- Modify: `tools/check_release_identity.ps1`
- Modify: `tools/check_release_security.ps1`

**Interfaces:**
- Consumes: CMake 共享变量 `BLACKHOLE_VERSION`。
- Produces: Qt 应用版本 `1.2.1`、EXE `ProductVersion=1.2.1`、`FileVersion=1.2.1.0`、Release 标签 `v1.2.1`。

- [ ] **Step 1: 先把身份和安全检查期望改为 1.2.1**

将两个检查脚本中当前产品版本、文件版本、Release 信息和成功输出改为：

```text
ProductVersion: 1.2.1
FileVersion: 1.2.1.0
Version: 1.2.1
Tag: v1.2.1
```

- [ ] **Step 2: 运行检查并验证 RED**

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\check_release_identity.ps1
```

Expected: FAIL，旧 `cmake/AppVersion.cmake` 仍为 `1.2.0` 或旧构建产物版本不匹配。

- [ ] **Step 3: 更新唯一版本源和发布元数据**

```cmake
set(BLACKHOLE_VERSION "1.2.1")
```

`package_release.ps1` 写入：

```powershell
"Version: 1.2.1"
"Tag: v1.2.1"
```

- [ ] **Step 4: 检查当前版本硬编码边界**

```powershell
rg -n "1\.2\.0" cmake package_release.ps1 tools
```

Expected: 当前版本身份文件中无 `1.2.0`；与版本身份无关的历史文档不在本任务修改范围。

- [ ] **Step 5: 提交版本同步源码**

```powershell
git add cmake/AppVersion.cmake package_release.ps1 tools/check_release_identity.ps1 tools/check_release_security.ps1
git commit -m "同步软件版本为v1.2.1"
```

### Task 2: 干净构建、身份验证与发布

**Files:**
- Generated: `release/`
- Generated: `BlakholeUI-v1.2.1-windows-x64.zip`
- Modify: `debug_state.md`（本地记录，不提交）

**Interfaces:**
- Consumes: Task 1 的 `1.2.1` 版本源和验证契约。
- Produces: 经过完整校验的 v1.2.1 Release ZIP。

- [ ] **Step 1: 正式干净构建和打包**

```powershell
$env:Path='C:\Qt\Tools\mingw1310_64\bin;C:\Qt\6.11.1\mingw_64\bin;' + $env:Path
powershell -NoProfile -ExecutionPolicy Bypass -File .\package_release.ps1
```

Expected: Renderer 与 Qt UI 构建成功，输出 `Release package ready`。

- [ ] **Step 2: 验证 EXE 与 Release 身份**

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File tools\check_release_identity.ps1
powershell -NoProfile -ExecutionPolicy Bypass -File tools\check_release_security.ps1 -ReleaseDir .\release
powershell -NoProfile -ExecutionPolicy Bypass -File tools\verify_release_checksums.ps1 -ReleaseDir .\release
```

Expected: `RELEASE_IDENTITY_OK version=1.2.1`、`RELEASE_SECURITY_OK version=1.2.1`、`RELEASE_CHECKSUMS_OK`。

- [ ] **Step 3: 运行更新状态和诊断回归**

```powershell
cmake --build Blakhole_UI\build\Desktop_Qt_6_11_1_MinGW_64_bit-Release --target update_release_tests update_checker_state_tests renderer_startup_diagnostics_tests
ctest --test-dir Blakhole_UI\build\Desktop_Qt_6_11_1_MinGW_64_bit-Release --output-on-failure
powershell -NoProfile -ExecutionPolicy Bypass -File tools\check_renderer_diagnostics.ps1
```

Expected: 测试通过，历史 `v1.2.0` 更新状态样例继续有效。

- [ ] **Step 4: 生成并核对 v1.2.1 ZIP**

```powershell
Compress-Archive -Path .\release\* -DestinationPath .\BlakholeUI-v1.2.1-windows-x64.zip -CompressionLevel Optimal -Force
Get-FileHash .\BlakholeUI-v1.2.1-windows-x64.zip -Algorithm SHA256
```

ZIP 必须包含两个 EXE、两张收款码、`RELEASE_INFO.txt`、校验清单，且不包含 `blackhole_debug.txt`。

- [ ] **Step 5: 推送并核对 SHA**

```powershell
git push origin main
$local=(git rev-parse main).Trim()
$remote=(git ls-remote --heads origin main).Trim().Split()[0]
if ($local -ne $remote) { throw "origin/main SHA mismatch" }
```
