# v1.2.3 设置保存修复实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 修复黑洞变换秒数输入和多屏模式持久化，并发布 v1.2.3。

**Architecture:** 保持 v4 配置格式不变，只修复 QML 小数转换和 C++ 行末字段解析。版本继续由 `cmake/AppVersion.cmake` 作为唯一来源，正式包使用短构建目录 clean build。

**Tech Stack:** Qt 6/QML、C++17、PowerShell、CMake/MinGW。

## Global Constraints

- 秒数范围保持 `0.5-60.0`，步进 `0.2`。
- 旧配置没有多屏字段时保持主屏默认值。
- 收款码只进入本地发布包，不进入 Git。
- 不覆盖用户当前未提交配置。

### Task 1: 设置行为回归修复

**Files:**
- Create: `tools/check_v123_settings_fixes.ps1`
- Modify: `Blakhole_UI/pages/BlackholeConfig.qml`
- Modify: `Blakhole_UI/core/blackholecore.cpp`

**Interfaces:**
- Consumes: `BlackHoleCore.slotSeconds`、`BlackHoleCore.screenTarget` 和 v4 配置第二行。
- Produces: 可编辑十分之一秒 SpinBox、可靠的行末 `screenTarget` 加载。

- [ ] **Step 1: 写失败契约**

契约要求 QML 包含“黑洞变换秒数”和 `valueFromText` 的乘十转换；C++ 在 `ls >> dm` 后使用流状态赋值，并禁止旧的 `if (!ls.atEnd()) m_screenTarget = dm`。

- [ ] **Step 2: 运行 RED**

Run: `powershell -File tools/check_v123_settings_fixes.ps1`
Expected: `V123_SETTINGS_FIXES_CHECK_FAILED`。

- [ ] **Step 3: 最小修复**

QML 增加：

```qml
valueFromText: function(text, locale) {
    var parsed = Number.fromLocaleString(locale, text)
    return isNaN(parsed) ? slotSpin.value : Math.round(parsed * 10)
}
```

C++ 在读取 `dm` 后使用 `ls.status() == QTextStream::Ok` 决定是否赋值。

- [ ] **Step 4: 运行 GREEN 与现有契约**

Run: `powershell -File tools/check_v123_settings_fixes.ps1`
Run: `powershell -File tools/check_qml_core_bindings.ps1`
Expected: 两项均通过。
