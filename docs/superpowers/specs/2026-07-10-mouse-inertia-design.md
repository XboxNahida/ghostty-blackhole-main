# 鼠标跟随惯性设计规格

## 目标
让“跟随鼠标移动”不再硬锁鼠标，而是可通过一个滑块控制跟随惯性。惯性为 0 时黑洞中心贴住鼠标；惯性大于 0 时黑洞有追赶滞后，并在鼠标附近做小范围游走。

## 用户行为
- 高级设置页新增“鼠标惯性”滑块。
- 滑块范围为 `0.00 ~ 1.00`，默认值为 `0.30`。
- `0.00` 表示无惯性、无游走，黑洞中心严格贴住鼠标位置。
- `1.00` 表示最大惯性，跟随速度最慢，附近游走半径最大。
- 滑块仅在“跟随鼠标移动”开启时显示，关闭跟随时该值仍保存但不参与渲染。

## 配置格式
继续使用 `blackhole_advanced.txt` 文本格式，新增字段:

```text
mouseInertia=0.30
```

兼容规则:
- 旧配置没有 `mouseInertia` 时使用默认值 `0.30`。
- 读取时将值限制在 `0.00 ~ 1.00`。
- 保存时保留两位小数。

## 渲染逻辑
在 C++ 渲染循环中处理跟随目标，不增加 shader 配置复杂度。

每帧流程:
1. 读取鼠标全局坐标。
2. 转换为当前黑洞窗口的 `0..1` 坐标。
3. 若 `mouseInertia <= 0`，直接使用鼠标坐标。
4. 若 `mouseInertia > 0`，用指数平滑让当前位置追赶鼠标。
5. 当有惯性时，额外叠加一个由时间和随机相位驱动的小范围椭圆游走偏移。
6. 最终坐标裁剪到 `0..1` 后写入 `uHomeX/uHomeY`。

建议参数:
- 跟随系数 `followAlpha = 0.42 - 0.36 * mouseInertia`，范围约 `0.06..0.42`。
- 游走半径 `wanderRadius = 0.035 * mouseInertia`。
- 游走频率使用当前已有 `phaseOffset` 参与，避免每次启动完全一致。

## 测试与验证
- 新增静态检查脚本，确认 UI、Qt Core、渲染器配置、渲染循环都接入 `mouseInertia`。
- 静态检查先红灯复现，再实现转绿。
- 构建主渲染器和 Qt UI。
- 执行 Release 打包脚本，确认关键文件存在。

## 非目标
- 不做运行时热更新。
- 不拆成“惯性”和“游走半径”两个滑块。
- 不修改 shader 中心锁定语义，shader 仍通过 `uFollowMouse` 决定是否使用 `uHomeX/uHomeY`。
