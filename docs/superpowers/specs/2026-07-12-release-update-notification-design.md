# Release 更新提醒设计

## 目标

让已下载旧版本的用户能够获知 GitHub 上的新正式版本，同时不阻塞程序启动、不主动弹窗打断用户，也不在程序内部下载或覆盖安装文件。

## 发布来源

- 最新版本来源固定为仓库 `XboxNahida/ghostty-blackhole-main` 的 GitHub Releases。
- 只检测正式 Release，不检测 Draft 和 Pre-release。
- Release 标签使用语义版本格式，例如 `v1.2.0`。
- 程序版本、关于窗口版本和更新比较必须使用同一个构建版本来源，避免多处硬编码不一致。

## 自动检查

- 每次程序启动完成后异步检查一次，不延迟主窗口显示、空闲检测或黑洞渲染启动。
- 自动检查只更新内部状态和设置按钮红点，任何结果都不主动弹窗。
- 发现高于当前程序版本的正式 Release 时显示红点。
- 没有更新时清除过期状态和红点。
- 自动检查遇到断网、超时、GitHub 限流或响应格式错误时静默结束，保留当前程序功能。

## 设置入口

- 设置按钮右上角增加稳定尺寸的小红点，仅在存在需要关注的新版本时显示。
- 设置抽屉增加“检查更新”选项，显示当前版本和简短状态，例如“发现 v1.2.0”“当前已是最新版本”或“检查失败”。
- 点击“检查更新”时发起新的异步请求，并显示检查中的状态，不能阻塞界面。

## 手动检查弹窗

- 只有用户点击“检查更新”后才显示结果弹窗。
- 有新版时显示当前版本、最新版本、简要更新说明、“下载”和“忽略此版本”。
- 没有新版时显示“当前已是最新版本”。
- 请求失败时显示可理解的错误信息，并允许关闭后继续使用软件。
- 手动检查代表用户重新考虑更新，因此会撤销当前线上版本的“已忽略”和“已访问下载页”抑制状态；只要仍有新版，就重新显示更新弹窗。

## 下载与忽略语义

- 点击“下载”后使用系统默认浏览器打开该 Release 的 HTML 页面，不在程序内直接下载或覆盖文件。
- 下载页打开成功后记录“已访问版本”。后续自动检查仍保留红点，但不产生弹窗；自动检查本来也不主动弹窗。
- 点击“忽略此版本”后持久化该版本号，并清除当前版本对应的红点。
- 线上出现更高版本时，旧的忽略和已访问状态自动失效，新版本重新显示红点。
- 安装新版后，当前程序版本不低于线上版本时清理旧状态和红点。

## 技术方案

- 使用 Qt Network 的异步网络请求访问 GitHub Releases 最新正式版接口。
- 请求携带明确的 `User-Agent`，设置有限超时，并限制响应体大小。
- 解析 `tag_name`、`name`、`body` 和 `html_url`，不执行响应中的任何内容。
- 使用严格的语义版本比较，忽略标签开头的 `v`，不使用字符串字典序比较。
- 网络层只负责获取和解析版本信息；状态机负责当前版本、忽略版本、已访问版本和红点；QML 只负责展示与用户命令。

## 状态流

1. 启动自动检查发现无新版：状态为最新，红点关闭。
2. 启动自动检查发现未忽略的新版：状态为可更新，红点开启，不弹窗。
3. 启动自动检查发现已忽略的同一版本：状态为已忽略，红点关闭。
4. 启动自动检查发现已访问下载页的同一版本：状态仍为可更新，红点开启，不弹窗。
5. 用户手动检查：清除当前线上版本的抑制状态并展示结果弹窗。
6. 用户点击下载：打开浏览器并记录已访问版本，红点保留。
7. 用户点击忽略：记录忽略版本，关闭弹窗并清除红点。
8. 发布更高版本：旧状态失效，新版本按可更新状态处理。

## 错误处理

- 自动请求失败不弹窗、不反复重试、不影响启动。
- 手动请求失败在设置区域或弹窗内显示原因。
- GitHub 返回空版本号、非法 URL 或无法解析的版本时视为检查失败，不显示虚假更新。
- 外部浏览器打开失败时保留弹窗和红点，并显示失败提示。
- 不保存 GitHub 响应中的可执行内容，不接受响应提供的本地文件路径。

## 发布指南

新增中文发布指南，面向不熟悉 Git/GitHub 的维护者，至少包含：

1. 修改项目唯一版本号。
2. 完成 Release 编译、回归测试和打包。
3. 在 GitHub Releases 创建与程序版本对应的 `vX.Y.Z` 标签。
4. 填写中文标题和更新说明。
5. 上传完整 Windows Release 压缩包。
6. 确认未勾选 Pre-release，并发布正式 Release。
7. 从公开页面下载附件并在干净目录验证启动。
8. 说明 Draft、Tag、Pre-release 和正式 Release 的差别，以及误发布后的撤回方法。

## 验收与测试

- 单元测试覆盖语义版本比较，包括 `v1.10.0` 大于 `v1.9.9`、相等版本和非法标签。
- 使用本地构造响应测试 GitHub JSON 解析，不依赖实时网络完成自动化测试。
- 状态测试覆盖自动检查不弹窗、红点规则、手动检查撤销忽略、下载后红点保留、忽略后红点清除和更高版本重置状态。
- 静态 QML 契约检查覆盖设置红点、检查更新入口和弹窗按钮。
- 手工验证断网、请求超时、正式 Release、Draft 和 Pre-release 场景。
- 手工验证浏览器打开 Release 页面，程序不会自行覆盖文件。
- 完成 Qt UI Release 全量构建，确认无新增错误和警告，并运行既有回归检查。

## 不在本次范围

- 不自动下载、解压、安装或替换程序文件。
- 不检测 Draft 和 Pre-release。
- 不建立独立更新服务器或遥测系统。
- 不要求用户登录 GitHub，也不在程序中保存 GitHub 凭据。
