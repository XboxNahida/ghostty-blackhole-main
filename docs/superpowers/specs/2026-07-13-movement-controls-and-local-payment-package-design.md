# 黑洞运动控制与本地收款资源打包设计

## 目标

本次修改解决两个独立问题：

1. 高级设置中的“路径随机化”和“动画速度”语义不准确，无法直接控制黑洞出现位置和自由移动速度。
2. 公开源码不得包含收款码图片，但正式 Release ZIP 需要保留关于页的收款说明与两张本地收款码。

同时，高级设置选项增加后必须支持滚动，且不显示可见滚动条。

## 运动设置界面

高级设置页的运动行为区域删除“路径随机化”复选框，将其替换为“黑洞出现位置”下拉框。下拉框提供以下五个选项：

- 随机
- 左上
- 右上
- 左下
- 右下

原“动画速度”滑块改为“黑洞移动速度”，范围为 `0.1x` 至 `3.0x`，步进为 `0.1x`，默认值为 `1.0x`。该参数只控制未启用鼠标跟随时的自由移动轨迹速度，不改变吸积盘自身旋转速度，也不参与鼠标惯性、过冲或鼠标坐标计算。

高级设置页使用 `Flickable` 承载原有内容。仅在内容超过可视高度时允许鼠标滚轮、触控板和触摸滚动，不创建 `ScrollBar` 或 `ScrollView`，页面上不显示滚动条。

## 配置模型

新增两个语义明确的配置字段：

- `spawnPosition`：整数枚举，`0` 为随机，`1` 为左上，`2` 为右上，`3` 为左下，`4` 为右下。
- `movementSpeed`：浮点倍率，保存和加载时限制在 `0.1` 至 `3.0`。

Qt Core、QML 和原生 Renderer 使用同一字段名和取值范围。默认配置为随机出现、`1.0x` 移动速度。

旧配置继续兼容：

- 仅存在 `randomPath=1` 时迁移为 `spawnPosition=0`。
- 仅存在 `randomPath=0` 时迁移为 `spawnPosition=2`，对应旧实现的右上出生点。
- 旧 `animationSpeed` 和 `spd` 继续按原规则读取为吸积盘动画速度，不能迁移为黑洞移动速度，避免改变已有用户的视觉预设。
- 保存新配置时写入 `spawnPosition` 和 `movementSpeed`，不再写入 `randomPath`。

## 渲染行为

随机出现模式保持当前随机出生坐标、随机轨迹相位和随机预设偏移。四个固定位置使用屏幕内安全坐标，避免黑洞中心直接落到屏幕边缘：

- 左上：`(0.18, 0.18)`
- 右上：`(0.82, 0.18)`
- 左下：`(0.18, 0.82)`
- 右下：`(0.82, 0.82)`

固定位置只决定出生点。自由移动轨迹仍然运行，轨迹相位和预设偏移采用稳定默认值，避免每次启动结果不一致。

自由移动时，将传给 Shader 的轨迹时间改为 `elapsedTime * movementSpeed`。黑洞生长、出生/退出动画、更新检测和空闲检测仍使用真实经过时间，不随移动速度改变。

启用鼠标跟随时，黑洞中心继续由鼠标坐标和 `mouseInertia` 计算，`movementSpeed` 不参与鼠标目标、速度、惯性或过冲限制。允许保留极轻微的惯性游走时，该游走也沿用鼠标跟随时间基准，不使用自由移动速度倍率。

## 收款资源与公开源码

公开源码继续遵守以下限制：

- Git 不跟踪 `QR_payment.jpg` 和 `WeChat_QR.png`。
- `src.qrc` 不嵌入收款码。
- 公开 QML 不直接引用收款码文件名或本地绝对路径。

关于页新增可选的本地收款区域。QML 通过 Qt Core 暴露的本地文件 URL 属性加载 Release 目录中的两张图片；图片不存在时，整个“请作者喝咖啡”区域隐藏，不显示损坏占位图。这样公开源码可正常构建和运行，正式本地 ZIP 则显示收款内容。

正式打包脚本从以下未跟踪本地路径读取图片：

- `Blakhole_UI/fonts/pic/QR_payment.jpg`
- `Blakhole_UI/fonts/pic/WeChat_QR.png`

打包时复制到 Release 的 `fonts/pic/` 目录，并在生成校验清单前确认两张图片存在。缺少任意图片时，正式打包失败并给出明确路径。图片只进入本地 `release` 和 ZIP，不进入 Git 提交。

## 验证

实施时按测试优先完成以下验证：

- QML 不再出现“路径随机化”和“动画速度”，并包含出现位置下拉框、移动速度滑块及无滚动条 `Flickable`。
- Qt Core 和 Renderer 对 `spawnPosition`、`movementSpeed` 的默认值、范围和保存加载一致。
- 旧 `randomPath` 配置正确迁移，旧 `animationSpeed`/`spd` 不被错误迁移。
- 四个固定出生位置和随机模式映射正确。
- 自由移动时间应用速度倍率，鼠标跟随坐标链路不引用 `movementSpeed`。
- 公开跟踪文件、QRC 和公开 QML 中没有收款码图片资源。
- Release 和最终 ZIP 中存在两张收款码，校验清单包含它们。
- Renderer、Qt UI、现有 Qt 测试、鼠标跟随契约和发布安全检查全部通过。

## 范围外事项

本次不调整鼠标惯性物理模型、不改变吸积盘旋转速度、不修改预设文件结构，也不移动或重建已经发布的 Git 标签。
