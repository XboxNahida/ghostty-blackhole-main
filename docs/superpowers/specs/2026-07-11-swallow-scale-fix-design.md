# 吞噬模式采样尺度修复设计

## 背景

当前吞噬模式会把大范围桌面卷成一个巨型球面。根因不是吸积盘预设，而是额外旋入偏移直接使用屏幕坐标常量，同时广域 UI 擦除参与整片颜色替换。两者都没有受到黑洞半径约束。

## 目标

- 关闭吞噬模式时保持原有渲染结果。
- 开启吞噬模式时，远场只表现为连续、可识别的弱透镜扭曲。
- UI 只在吸积盘和事件视界附近被拉伸、旋入和去色。
- 黑洞大小改变时，旋入范围随 `rh` 同比例变化，不再出现固定屏幕尺寸的玻璃球。
- 保留现有程序化吸积盘亮度和事件视界黑化。

## 方案

### 采样层

保留 `noShellLensP` 作为单次远场透镜采样。将切向旋入和径向内拉先计算为无量纲强度，再统一乘以基于 `rh` 的 `infallDisplacementScale`。这样最大位移由黑洞尺寸决定，而不是接近整个屏幕高度。

继续使用连续的 `adaptiveCollapse`，但两端采样坐标之间只允许小尺度差异。该混合负责让近场逐渐进入吸积盘流动，不再承担大范围桌面搬移。

### 颜色层

`nearFieldUiCutoff` 继续负责事件视界和吸积盘附近的 UI 切断。`wideTidalErase` 只用于远场轻度去饱和，不再直接把大范围像素替换成吸积盘颜色。

实际发光仍来自 `emitc`、`trans`、`captured` 和预设参数，不新增固定形状的彩色光圈。

## 验证

1. 静态检查必须拒绝未按 `rh` 缩放的 `orbitalInfall`。
2. 静态检查必须拒绝 `wideTidalErase` 直接主导 `uiSuppression`。
3. 主程序和 Qt UI Release 构建通过。
4. 运行时 shader 编译、链接通过。
5. Release 重新打包，供同一桌面场景复测。

## 风险

- 旋入幅度降低后，近场流动会比当前温和，但仍保留方向和时间变化。
- 单帧 shader 不能保存真实物质历史轨迹；本次只修复采样尺度和圆形边界，不引入历史缓冲。

## 视觉复测后的架构修正

第一次尺度修复后，截图中的大圆边界仍然存在。复查基础 shader 后确认，边界对应 `b == bmax` 的远场解析透镜与近场光线追踪交界。

基础 shader 专门拟合了两侧位移，使交界误差低于 1%。吞噬注入只放大近场终端采样，远场分支仍按原公式提前返回，破坏了这项连续性。旋入位移过大只是次要问题，不是圆边界主因。

修正后的不变量如下：

- 远场和近场基础透镜必须使用相同的 `sharedLensBoost` 公式。
- `longTailLens` 和 `edgeFreeWarp` 不能只存在于近场分支。
- `orbitalInfall` 与额外坍缩必须乘 `handoffFade`，且在 `b == bmax` 时严格收敛到 0。
- 颜色层在交界处必须保持 `uiSuppression == 0`，不改变远场提前返回的颜色。
