# v1.2.2 帧率限制与杀毒误报缓解设计

## 目标

发布 `v1.2.2`，完成两项改进：

1. 在高级设置中增加桌面 Renderer 帧率限制，降低常驻黑洞的 GPU 占用和功耗。
2. 在不签名、不上传多引擎扫描、不逐厂商申诉的前提下，继续通过合法身份元数据、隔离场景诊断和发布校验信息缓解杀毒误报带来的使用问题。

本版本不承诺消除全部安全软件误报，不尝试关闭安全软件、写入排除项、恢复隔离文件或绕过系统安全策略。

## 当前基础

### 渲染循环

- `src/win32_gl.cpp` 创建 OpenGL 上下文后调用 `wglSwapIntervalEXT(1)`，当前由垂直同步决定帧率。
- `src/main.cpp` 每帧使用 `Win32GL_GetTime()` 计算真实时间，Shader 动画、自由移动和退出动画不依赖累计帧数。
- 当前没有用户可配置的帧率上限。

### 既有误报缓解

- 两个 EXE 已包含版本资源，并以普通用户权限运行。
- PE 已保留 ASLR、高熵地址和 DEP/NX，不使用 UPX、加壳、混淆或伪造签名。
- Renderer 不包含网络通信功能。
- Renderer 已取消按名称终止任意进程，使用 Job Object 管理自有子进程。
- 全局按键轮询已改为 `RegisterHotKey`。
- Release 已包含 `RELEASE_INFO.txt`、`release_checksums.sha256`、校验脚本和 `SECURITY.md`。

剩余主要风险是未签名新文件的低信誉，以及桌面捕获、进程枚举、子进程管理、全局热键和开机启动等合法行为的组合启发式特征。

## 帧率设置模型

### 数据表示

新增整数配置 `frameRateLimit`：

- `0`：无限制。
- `30–240`：目标 FPS。
- 其他输入：规范化为安全值。
  - 小于 `30` 且不为 `0` 时按 `30` 处理。
  - 大于 `240` 时按 `240` 处理。
- 默认值：`60`。

旧版 `blackhole_advanced.txt` 没有该字段时自动使用 `60`，不修改其他高级设置。

### Qt/Core 接口

`BlackHoleCore` 增加：

- `Q_PROPERTY(int frameRateLimit READ frameRateLimit WRITE setFrameRateLimit NOTIFY frameRateLimitChanged)`。
- `int frameRateLimit() const`。
- `void setFrameRateLimit(int value)`。
- `void frameRateLimitChanged()`。
- 成员默认值 `m_frameRateLimit = 60`。

保存到 `blackhole_advanced.txt`：

```text
frameRateLimit=60
```

恢复默认设置时重置为 `60`。Setter 只更新配置和信号，不直接控制已运行的 Renderer；和现有高级设置一致，下次启动 Renderer 时生效。

### QML 滑块

高级设置的捕获与显示区域新增“帧率限制”：

- 滑块范围 `30–241`，步进 `1`。
- `30–240` 显示为对应的 `FPS`。
- UI 值 `241` 显示“无限制”，写入 Core 时转换为 `0`。
- Core 值 `0` 映射回 UI 的 `241`。
- 默认显示 `60 FPS`。
- “无限制”状态显示简短的高 GPU 功耗提示。

为避免在页面中复制整个滑块组件，`ESlider.qml` 增加可选的自定义数值文本属性。未设置时仍使用现有 `prefix + value.toFixed(decimals) + suffix`，保证其他滑块外观和行为不变。

帧率限制只作用于桌面 `blackhole.exe --render`。Qt 实时预览仍由 Qt Quick 渲染循环管理，不受该滑块影响。

## Renderer 帧率调度

### 配置读取

`BlackholeConfig` 增加 `int frameRateLimit = 60`。`SaveAdvancedConfig()` 和 `LoadAdvancedConfig()` 增加同名键，并通过统一的规范化函数处理 `0`、`30–240` 和越界输入。

Renderer 启动日志记录最终值：

```text
[Init] Frame rate limit: 60 FPS
```

无限制时记录：

```text
[Init] Frame rate limit: unlimited
```

### 垂直同步

当前固定的 `wglSwapIntervalEXT(1)` 改为显式可控：

- 用户设置 `30–240` 时关闭 WGL 垂直同步，由软件限帧控制目标 FPS。
- 用户设置 `0` 时同样关闭垂直同步，并且不做软件等待。
- 如果 `WGL_EXT_swap_control` 不可用，软件限帧仍正常工作；日志记录无法设置交换间隔，但不视为启动失败。

这样可以在高刷新率显示器上真正限制为 60 FPS，也让“无限制”具有明确含义。

### 限帧器

新增一个小型、无 Qt 依赖的帧率调度单元，负责：

- 规范化配置值。
- 根据 `QueryPerformanceCounter` 时间推进下一帧截止时间。
- 在有帧率限制时等待至截止时间。
- 渲染耗时超过一个周期时跳过过期截止点，不累计越来越大的延迟。
- 无限模式立即返回，不等待。

等待优先使用 Windows 高精度 Waitable Timer；创建失败时回退到基于高精度时钟和短暂 `Sleep` 的等待。限帧器不修改 Shader 时间、鼠标物理或捕获时间基准。

帧率等待放在 `SwapBuffers` 之后、下一帧消息处理之前。退出渐隐仍使用真实时间，因此低帧率不会延长退出时长。

## 低成本误报缓解

### 身份元数据

两个 VERSIONINFO 资源继续使用统一版本源，并补充：

- 官方源码仓库 URL。
- 清晰的本地桌面渲染/配置说明。
- `v1.2.2` 文件版本和产品版本。

不伪造公司、证书或微软签名，不修改 PE 节区名称，不加入随机填充。

### Renderer 被隔离的诊断

启动预检继续检查：

- `blackhole.exe`。
- `shaders/vert.glsl`。
- `shaders/frag_desktop_header.glsl`。
- `blackhole.glsl`。

如果缺少的是 `blackhole.exe`，错误摘要改为针对性说明：

```text
渲染器文件 blackhole.exe 缺失。请确认已完整解压；如果文件解压后自动消失，可能被安全软件隔离。
```

详细信息列出实际缺失路径，并建议只在确认官方 Release 来源和 SHA-256 后恢复或单独放行，不要求关闭全部防护。

如果缺少的是 Shader，继续显示普通“渲染器所需文件缺失”，避免把所有文件缺失都误判为杀毒隔离。

错误窗口保留“复制错误详情”和打开目录能力。缺文件时打开 Renderer 工作目录，即用户的解压目录；程序不尝试打开、控制或修改任何第三方安全软件。

### 发布透明度

最终包生成：

- `BlakholeUI-v1.2.2-windows-x64.zip`。
- `BlakholeUI-v1.2.2-windows-x64.zip.sha256.txt`。

Sidecar 文件包含 ZIP 的大写 SHA-256 和文件名，上传 Release 时与 ZIP 一起发布。ZIP 内部继续包含完整文件清单和校验脚本。

继续执行 `strip.exe --strip-debug` 的现有策略：它只移除 Release 副本的调试节，不做压缩、加壳或混淆。本版本没有多引擎 A/B 证据，不凭感觉切换构建策略。

## 版本同步

唯一版本源改为：

```cmake
set(BLACKHOLE_VERSION "1.2.2")
```

同步范围包括：

- Renderer 和 Qt UI VERSIONINFO。
- Qt 应用版本与更新检查当前版本。
- `package_release.ps1` 的 `Version` 和 `Tag`。
- 发布身份、安全和打包检查脚本的预期版本。
- README 当前版本、下载文件名和 Release 链接。
- 发布指南示例中的当前版本。

历史更新解析测试中的旧版本样例不做机械替换。

## 测试与验证

### 配置与 UI

- 先增加失败测试，覆盖默认 `60`、`0`、`30`、`240` 和越界输入。
- 验证 `blackhole_advanced.txt` 保存/加载 `frameRateLimit`。
- 验证旧配置缺少字段时使用 `60`。
- 静态契约验证 QML 滑块、`0 ↔ 241` 映射、“无限制”文本和 Core 绑定。
- 验证 `ESlider` 未提供自定义文本时保持原有格式。

### Renderer

- 纯逻辑测试覆盖截止时间推进、落后重置和无限模式不等待。
- 静态契约验证固定 `wglSwapIntervalEXT(1)` 被移除，Renderer 使用规范化后的帧率配置。
- 启动测试确认日志分别输出 `60 FPS` 和 `unlimited`。
- 手动运行 `30`、`60`、`240` 和无限制，观察窗口 FPS、GPU 占用和退出动画；帧率允许受渲染性能限制，不能把硬件达不到目标值视为限帧器失败。

### 隔离诊断

- 状态机测试覆盖 `blackhole.exe` 缺失时的专用提示。
- Shader 缺失仍走普通文件缺失提示。
- 错误详情包含实际路径，单次启动不重复弹窗。

### 发布

- Renderer 与 Qt UI `--clean-first` Release 构建。
- 运行全部 CTest 和现有静态/安全契约。
- 验证两个 EXE 均为 `ProductVersion=1.2.2`、`FileVersion=1.2.2.0`。
- 验证 `RELEASE_INFO.txt` 为 `Version: 1.2.2`、`Tag: v1.2.2`。
- 验证正式 ZIP 包含本地收款码，但 Git 提交不包含收款码或 ZIP。
- 独立解压 ZIP，启动 UI 并测试 Renderer。
- 重新计算 ZIP SHA-256，验证 sidecar 内容完全一致。
