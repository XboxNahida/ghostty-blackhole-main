# 吸积盘高质量光影效果设计

## 目标

将现有“吞噬屏幕 UI 特效”完整替换为单一的“吸积盘光影效果”开关。开启后，所有黑洞预设都应具有接近参考图的分层吸积盘、明暗纹理、双色照明和强烈辉光；关闭后维持原始黑洞渲染与原有性能。

本功能不提供强度滑块。实现优先保证光效质量，允许开启时增加 GPU 渲染开销。

## 视觉结构

光影由下列层次共同组成：

1. 外层光雾：沿吸积盘实际投影向外宽范围衰减，颜色较淡，不形成硬边界。
2. 中层亮带：依赖现有 `emitc`、`trans` 和盘面投影信息强化真实发光区域，形成白金色高亮主体。
3. 暗纹层：在吸积盘自身坐标中生成非均匀、随时间缓慢流动的黑色条纹。暗纹只压低局部发光，不切断盘面连续性。
4. 双色照明：在吸积盘自身坐标中划分冷暖区域，一侧为金色，另一侧为冰蓝色。该方向随每个预设的倾角、滚转和盘面运动变化，不固定在屏幕左右。
5. 光子环：在事件视界外缘形成细窄白金高光，事件视界内部继续保持纯黑。
6. Bloom：提取高亮区域，经多次模糊后与原始画面合成，形成参考图中的强光外溢和爆发感。

所有遮罩必须使用连续过渡，并受实际盘面或光子环数据约束，禁止重新引入吞噬模式曾出现的圆形采样边界。

## 渲染架构

默认 OpenGL 路径增加离屏渲染流程：

1. 基础黑洞 shader 渲染到 HDR 颜色纹理，并在光影开关开启时加入分层光源。
2. 从 HDR 结果中提取高亮部分。
3. 使用横向、纵向分离的高斯模糊执行多轮 ping-pong 处理。
4. 将模糊后的辉光与原始 HDR 颜色合成并输出到窗口。

关闭光影时跳过亮部提取和模糊流程，直接使用基础结果，避免无意义的额外开销。

Bloom 负责宽范围柔光，吸积盘渐变、暗纹和双色方向由基础 shader 负责。两者职责分离，避免用模糊掩盖盘面结构。

## 配置与界面

- 高级设置仅显示“吸积盘光影效果”勾选框。
- 删除吞噬强度滑块。
- 新配置字段命名为 `lightingEffect`，默认关闭。
- 读取旧配置时，将 `screenSwallow` 作为一次兼容输入映射到 `lightingEffect`。
- 旧 `swallowStrength` 可读取但忽略，不再参与任何渲染或运动计算。
- 保存配置时只写入 `lightingEffect`，不再写吞噬字段。

旧吞噬 shader 注入、桌面 UI 去色、采样旋入、近场压暗、运行速度降低和运动范围缩减逻辑全部删除。删除前通过静态搜索确认调用关系，并由新的回归检查保证旧逻辑不再残留。

## 预设适配

光影不绑定某个预设编号或固定半径。每帧从当前预设读取温度、倾角、滚转、内外盘半径、亮度和曝光参数，并结合现有光线追踪结果生成遮罩。因此顺序、循环、随机播放以及预设交叉淡入时都使用同一套光影算法。

双色方向以盘面坐标计算，随 `L.roll` 和 `L.incl` 变化。极低倾角、极高倾角和窄盘预设需要保持连续，不能因投影宽度接近零产生除零、NaN 或整屏发光。

## 实施阶段

由于预计修改超过 3 个文件，按以下阶段实施并分别验证：

1. 配置与 UI：新增光影布尔配置，完成旧字段兼容读取，删除强度控件和吞噬配置输出。
2. 分层光源：先增加失败检查，再删除吞噬逻辑并实现外层光雾、中层亮带、暗纹、双色照明与光子环。
3. Bloom：增加 HDR 帧缓冲、亮部提取、分离模糊和最终合成，并处理窗口尺寸变化与资源释放。
4. 验证与打包：运行静态检查、shader 编译短测、主程序和 Qt UI Release 构建，并对多个不同姿态预设进行截图复核。

每个阶段完成后立即更新 `debug_state.md`。

## 验收标准

- 界面中只有一个“吸积盘光影效果”勾选框，不存在吞噬或强度调节文案。
- 关闭开关时不执行 Bloom，原始黑洞外观不因本功能改变。
- 开启开关后，至少验证正面、倾斜和不同滚转方向的多个预设均有外淡、中亮黑纹、金蓝双色和柔和辉光。
- 金蓝方向随盘面姿态变化，而不是固定在屏幕坐标。
- 事件视界保持黑色，光影没有覆盖中心黑洞。
- 画面不出现吞噬、桌面 UI 被卷入、硬圆边界、全屏过曝或 NaN 闪烁。
- 主程序和 Qt UI Release 构建无新增错误；新增警告必须说明。

## 风险与边界

- 参考图属于概念效果图，无法保证逐像素一致；验收目标是结构、色彩关系和辉光强度接近，而非复刻其具体纹理。
- 多轮 Bloom 会增加 GPU 占用，尤其在高分辨率和多显示器模式下。只在开关开启时启用，并优先保证渲染正确。
- 默认 D3D11 渲染路径当前处于实验状态，本阶段以项目实际默认的 OpenGL 路径为实现和验收范围，不扩展未启用的实验路径。
