# README 首页重构设计

## 目标

将仓库根目录 `README.md` 从偏开发记录的技术说明，重构为普通用户进入 GitHub 项目首页后可以直接完成下载、安装、使用和故障反馈的中文入口，同时为开发者保留简洁、准确的构建信息。

当前发布版本统一展示为 `v1.2.1`，下载入口指向 GitHub Releases 页面：

`https://github.com/XboxNahida/ghostty-blackhole-main/releases`

## 受众与信息优先级

首页优先服务第一次接触软件的 Windows 用户，信息顺序如下：

1. 这是什么软件，以及实际效果演示。
2. 从哪里下载当前版本。
3. 软件能做什么。
4. 如何正确解压和启动。
5. 当前兼容范围与检测边界。
6. 遇到无黑洞、DXGI 初始化失败或杀毒误报时如何处理。
7. 如何提交有效的故障信息。
8. 开发者如何构建，以及更深入的技术文档在哪里。

不再把旧 UI 状态、开发进度和作者更新时间放在首页顶部。

## 页面结构

### 1. 首屏

- 使用项目名“Black Hole / 黑洞桌面特效”作为主标题。
- 保留 `demo.gif`，让效果成为首屏主要视觉信息。
- 展示当前版本、Windows 平台和 MIT License 徽章。
- 用一句话说明：软件捕获 Windows 桌面并实时渲染黑洞引力透镜、吸积盘和光子环效果。
- 提供醒目的 `v1.2.1` 下载链接和 Releases 页面入口。

### 2. 主要功能

用紧凑列表覆盖已发布能力：

- 始终显示和空闲触发两种运行模式。
- 视频播放与游戏运行检测。
- 视频提示、始终允许触发、前台强制不触发三类名单。
- 手动输入、运行程序选择和可执行文件选择三种名单添加方式。
- 主屏、副屏、跨屏和一屏一黑洞。
- 鼠标跟随、惯性、黑洞出现位置和自由移动速度。
- 预设、吸积盘光影和实时预览。
- 启动时非阻塞检查更新，并在设置入口显示红点；只有用户手动检查时弹出更新窗口。
- Renderer 启动失败诊断、错误详情复制和日志目录入口。

### 3. 快速开始

明确说明：

1. 从 Releases 下载完整 ZIP。
2. 完整解压，不要只复制单个 EXE。
3. 启动 `appBlakholeUI.exe`。
4. 配置后点击启动。
5. 通过系统托盘退出。

运行目录中的 Qt DLL、Shader、图片和 Renderer 都是必要文件。README 不再宣称单个 `blackhole.exe` 可以代表完整 Release 独立运行。

### 4. 兼容性

保留对用户有用的最低条件，删除过度细分且难以持续验证的显卡型号表：

- 64 位 Windows 10 1803 或更高版本。
- 推荐 Windows 11 22H2 或更高版本。
- 支持 OpenGL 3.3 的显卡和正常工作的显卡驱动。
- 自动捕获在 Windows 11 22H2+ 可优先使用无黄边 WGC；旧系统通常回退 DXGI。

不使用“完全支持所有显卡”等绝对表述。双显卡、远程桌面、虚拟机和第三方桌面捕获软件可能影响 DXGI Desktop Duplication。

### 5. 视频、游戏与名单说明

README 应准确说明检测能力和边界：

- 视频检测综合使用媒体会话、前台窗口和进程特征，包含对哔哩哔哩客户端等常用播放器的适配。
- 正在播放和暂停状态按当前软件规则处理；无声视频可能没有可用的系统媒体状态，不能保证识别。
- 游戏检测覆盖独占全屏、无边框和部分窗口化游戏，但低负载或特殊渲染方式的游戏可能漏检。
- 漏检时可将程序加入名单，名单支持从运行程序或可执行文件中选择，也保留手动输入。
- “前台强制不触发名单”只在目标程序位于前台时生效，避免后台进程永久阻止黑洞。

不承诺覆盖国内所有播放软件或所有游戏。

### 6. 常见问题与诊断

首页提供四类高频问题：

#### 启动后没有黑洞

- 确认完整解压，并检查 `blackhole.exe`、Qt 运行库和 Shader 文件是否存在。
- 检查当前运行模式；空闲模式需要达到设定时间，鼠标或键盘活动会隐藏黑洞。
- 查看程序弹出的 Renderer 错误详情，并使用“打开日志目录”。

#### `DXGI_Init primary failed`

- 说明该错误发生在桌面捕获初始化，而不是 Qt 窗口或 OpenGL 窗口创建阶段。
- 建议关闭 OBS、录屏、远程控制和动态壁纸软件后重试。
- 更新显卡驱动；双显卡设备可尝试让两个 EXE 使用同一块 GPU。
- Windows 10 用户可手动尝试 WGC，代价是可能出现黄色捕获边框。
- 远程桌面和虚拟机环境可能不支持 DXGI Desktop Duplication。

#### 杀毒软件误报

- 诚实说明项目包含桌面捕获、全屏置顶窗口、开机自启动、进程检测和联网更新检查，这些行为可能触发启发式检测。
- 建议只从本仓库 Releases 下载，并核对发布页提供的 SHA-256。
- 不要求用户关闭杀毒软件；无法放行时应停止运行并提交样本或检测名称。

#### 如何反馈问题

要求附带：

- `blackhole_debug.txt`。
- 必要时使用 `blackhole.exe --render 2> dxgi_error.txt` 获取的 `dxgi_error.txt`。
- Windows 版本、显卡型号、是否双显卡。
- 是否处于远程桌面或虚拟机。
- 是否同时运行录屏、远控或动态壁纸软件。

### 7. 更新机制

说明当前版本为 `v1.2.1`。软件启动时后台检查更新，只在设置入口显示红点；用户点击“检查更新”后才弹出下载或忽略版本选项，下载按钮会打开浏览器中的 Release 链接。

### 8. 开发者信息

- 保留 MSYS2 UCRT64、CMake、MinGW 和 Qt 6 的必要构建说明。
- 项目结构仅列出 `src/`、`Blakhole_UI/`、`shaders/`、`Doc/` 和 `release/` 的职责，不在首页展开完整源文件清单。
- 深入技术说明链接修正为 `Doc/TECHNICAL.md`。
- 保留 Eric Bruneton 黑洞着色器来源和 MIT License。

## 资源与隐私边界

- README 使用已跟踪的 `demo.gif`，不新增外部图片依赖。
- 不在 README 中嵌入两张本地收款码，也不将其加入 Git；收款码继续只进入本地发布 ZIP。
- 不提交 `BlakholeUI-v1.2.1-windows-x64.zip`。

## 验证标准

修改完成后必须验证：

1. README 中不存在“新功能还在开发”“旧 UI 仍可使用”“作者最近比较忙”等过时首页文案。
2. 当前版本、下载入口、程序入口和更新行为均与 `v1.2.1` 一致。
3. `demo.gif`、`Doc/TECHNICAL.md` 和 `LICENSE` 相对链接指向仓库中实际存在的文件。
4. 视频、游戏、DXGI 和杀毒误报说明不使用无法验证的绝对兼容承诺。
5. `git diff --check` 通过。
6. 提交中只包含 README、设计/计划文档和 `debug_state.md`，不包含 ZIP 与收款码。
