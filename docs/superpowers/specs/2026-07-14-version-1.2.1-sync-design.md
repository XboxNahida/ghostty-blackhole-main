# v1.2.1 版本同步设计

## 目标

将软件当前版本从 `1.2.0` 完整同步为 `1.2.1`，确保 Qt 界面、更新检查、两个 Windows EXE 的版本资源、Release 元数据、发布安全检查和最终 ZIP 文件名一致。

## 修改范围

- `cmake/AppVersion.cmake`：唯一版本源改为 `1.2.1`。
- `package_release.ps1`：Release 信息中的 `Version` 和 `Tag` 改为 `1.2.1`、`v1.2.1`。
- `tools/check_release_identity.ps1`：期望产品版本改为 `1.2.1`，文件版本改为 `1.2.1.0`。
- `tools/check_release_security.ps1`：Release EXE 和 `RELEASE_INFO.txt` 的期望版本改为 `1.2.1`。

不修改 `tests/update_checker_state_tests.cpp` 中的 `v1.2.0`，因为这些值用于模拟历史 Release、忽略状态和版本比较，不代表当前应用版本。

## 构建与发布

重新干净构建 Renderer 和 Qt UI，验证：

- Qt `applicationVersion` 为 `1.2.1`。
- `appBlakholeUI.exe` 与 `blackhole.exe` 的 `ProductVersion` 为 `1.2.1`。
- 两个 EXE 的 `FileVersion` 为 `1.2.1.0`。
- `RELEASE_INFO.txt` 写入 `Version: 1.2.1`、`Tag: v1.2.1`。
- Release 安全检查和完整校验清单通过。

最终生成 `BlakholeUI-v1.2.1-windows-x64.zip`。两张收款码继续进入本地 ZIP，但不被 Git 跟踪。

## 验收标准

- 源码中用于当前版本身份的 `1.2.0` 全部消除。
- 历史版本测试数据仍保留且测试通过。
- 两个 EXE、Release 信息和 ZIP 文件名均为 `1.2.1`。
- Release ZIP 不包含运行调试日志。
- 代码提交并推送到 `origin/main`，本地与远程 SHA 一致。
