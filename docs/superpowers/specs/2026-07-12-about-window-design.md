# Blakhole UI 关于窗口设计

## 目标

为 Blakhole UI 增加一个独立的“关于”窗口。用户可通过主页头像或设置抽屉中的“关于 Blakhole UI”入口打开窗口，查看软件简介、使用说明、待完善功能、GitHub 项目链接和作者收款码，并可在窗口内更换头像。

## 交互入口

- 点击主页左侧头像时打开关于窗口，不直接打开图片选择器。
- 点击设置抽屉底部“关于 Blakhole UI”时打开同一个关于窗口。
- 关于窗口只保留一个实例；窗口已经打开时，再次触发入口应将其显示、恢复并置于前台，不创建重复窗口。
- 设置入口应具有悬停和点击反馈，并在打开窗口后关闭设置抽屉。

## 窗口形式

- 使用独立 QML 子窗口，标题为“关于 Blakhole UI”。
- 默认尺寸约为 720 × 760 逻辑像素，并设置合理的最小尺寸。
- 窗口支持调整大小，主体使用纵向滚动区域。
- 窗口沿用主界面的深色或浅色主题，不建立独立主题配置。
- 关闭关于窗口时隐藏并保留该唯一实例，不影响主程序和黑洞渲染状态；再次打开时复用并置于前台。

## 内容布局

内容按以下顺序纵向排列：

1. 居中圆形头像。
2. 软件名称和版本信息。
3. 软件简介。
4. 基本使用说明。
5. “待完善功能：黑洞吞噬效果选项”提示。
6. 可点击的 GitHub 项目链接。
7. 普通分区标题“请作者喝咖啡”。
8. 支付宝和微信收款码。

收款码常驻显示，不使用折叠按钮。宽窗口中两个收款码并排，窗口变窄时自动切换为上下排列。用户通过滚动到达收款区域，初始视图仍以软件说明为主。

## 文案

软件简介：

> Blakhole UI 是一款 Windows 黑洞可视化工具，可根据空闲状态、媒体播放和游戏运行状态自动控制黑洞显示。

使用说明：

> 使用前可在主界面调整黑洞外观、动画和显示模式；需要自动触发时，请配置空闲时间及相关名单。遇到未识别的视频软件或游戏，可在“空闲名单”中手动添加。

待完善功能：

> 待完善功能：黑洞吞噬效果选项。该功能仍在开发和优化中，当前版本的相关行为可能不完整。

>
> 软件仍在持续完善，欢迎通过 GitHub 提交问题和建议。

## 颜色层级

- 软件名称、可点击链接和主要交互使用当前主题的强调色。
- 正文和使用说明使用当前主题正文色，并通过透明度区分主次。
- “待完善功能”使用柔和的提醒色和轻量背景，突出状态但不呈现为错误。
- “请作者喝咖啡”只使用普通分区标题尺寸和正文体系内的颜色，不使用超大字体、强烈动画或高对比募捐式设计。
- 支付宝和微信标签可分别使用克制的蓝色和绿色作为识别辅助，不能让整个页面被单一色系主导。

## 头像更换

- 关于窗口内的居中头像可点击，悬停时显示“点击更换头像”提示和轻微视觉反馈。
- 点击后打开本地图片选择器，支持 PNG、JPG、JPEG 和 WebP。
- 选择成功后，将图片复制到应用的用户配置目录，而不是长期引用原始文件路径，避免原文件移动或删除后头像失效。
- 自定义头像路径写入现有应用配置体系，并在下次启动时恢复。
- 主页头像和关于窗口头像绑定同一状态，选择后立即同步更新。
- 用户取消选择、文件无法读取或复制失败时保留原头像，不写入无效配置；失败原因显示在关于窗口内。
- 未设置自定义头像或自定义文件失效时，回退到内嵌的默认 `avatar.png`。

## 图片资源

- 默认头像继续使用 Qt 资源中的 `avatar.png`。
- `QR_payment.jpg` 和 `WeChat_QR.png` 加入 `src.qrc`，随程序嵌入并进入 Release 包，运行时不依赖源码目录。
- 收款码保持原始比例，完整显示，不裁切二维码有效区域。
- 显示尺寸使用稳定的响应式约束，窗口缩放时不遮挡文字或相互重叠。

## 组件边界

- 新增独立的关于窗口 QML 组件，负责窗口生命周期、滚动布局、内容展示、链接和头像点击入口。
- 扩展现有 `EAvatar`，增加可选点击信号、悬停状态和工具提示；默认行为保持不变，避免影响其他调用方。
- `BlackHoleCore` 负责自定义头像的选择、复制、持久化和有效路径暴露。QML 不直接处理文件复制或配置文件写入。
- `Main.qml` 只负责连接两个打开入口、共享头像状态和管理唯一关于窗口实例。

## 数据流

1. 程序启动时，Core 读取已保存的自定义头像路径并验证文件是否可读。
2. Main 和关于窗口从 Core 获取当前头像 URL；无有效自定义头像时使用内嵌默认头像。
3. 用户在关于窗口点击头像，QML 调用 Core 的头像选择入口。
4. Core 打开图片选择器，将有效图片复制到用户配置目录，保存路径并发出头像变化信号。
5. Main 和关于窗口同时收到变化并刷新图片。
6. 用户点击 GitHub 链接时，使用 Qt 的外部链接接口交给系统默认浏览器打开。

## 错误处理

- 重复打开关于窗口时复用现有窗口并置前，不产生多个窗口或重复信号连接。
- 图片选择取消视为正常操作，不显示错误。
- 图片读取、复制或配置保存失败时保留旧头像，并显示简短错误提示。
- 外部链接打开失败时不关闭关于窗口，可显示简短提示。
- 图片解码失败时回退默认头像，不允许空白头像导致布局变化。

## 验收与测试

- 静态契约测试先验证两个入口、唯一窗口实例、GitHub URL、头像选择入口和两个二维码资源引用；旧代码应按预期失败。
- Core 单元测试覆盖自定义头像路径规范化、有效文件复制、取消或失败时保持旧值、配置恢复和默认头像回退。
- QML 编译验证关于窗口、响应式布局和新增头像属性不存在语法或类型错误。
- 手工验证主页头像与设置入口均可打开同一个窗口，重复点击只激活已有窗口。
- 手工验证头像更换后两个位置立即同步，重启后仍保留；取消和无效图片不会破坏旧头像。
- 手工验证 GitHub 链接由默认浏览器打开。
- 手工验证深色、浅色主题，以及高 DPI、默认尺寸、最小尺寸下无文字截断、控件重叠和二维码裁切。
- 完成 Qt UI Release 全量构建，确认无新增错误和警告。
- 重新运行项目既有回归检查，并将验证后的程序重新打包、同步到根目录 `release`。

## 不在本次范围

- 不提供在线头像、账号系统或云同步。
- 不提供头像裁剪编辑器；图片使用现有圆形裁切显示方式。
- 不增加支付跳转、支付状态识别或收款统计。
- 不改变黑洞吞噬效果本身，只在关于窗口说明该选项仍待完善。
